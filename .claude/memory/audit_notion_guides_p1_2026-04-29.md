## 二、資料庫類別指南（`3279ff25fdab80a18fffff56c578a86a`）

### 🔴 致命（5 條）

#### 1. **DB04 大分類欄位「交接類型」實際 schema 中不存在**
指南第四節表格寫：
> DB04 共識交接協作｜交接類型｜專案協作 ／ 庫存門市

DB04 schema 沒有任何 `select` 欄位叫「交接類型」（已逐欄位掃描 215 個欄位）。最接近的是 `協作選項`（select：活動辦理／內容製作）、`協作參考`（multi_select）。
**建議**：要嘛在 DB04 新增「交接類型」select 欄位（選項：專案協作／庫存門市），要嘛把指南改為實際使用的欄位（建議用 `協作選項`，但選項值需重新對齊）。
※註：CLAUDE.md 標示「2026/04/28 DB04 交接類型刪掉共識互動」暗示此欄位曾經存在；可能已被整個刪除或改名。

#### 2. **DB05 大分類欄位「表單類型」選項與實際不符**
指南：`報名登記 ／ 共識互動 ／ 圖文影音`
實際 schema：`報名登記 ／ 共識互動 ／ 圖文影音` ✅ 正確 — **但 DB05 尚有未在指南列出的另一個分類欄位 `互動選項`（內部共識／外部聯繫）**。指南把 `表單類型` 描述成「最高層大分類」沒問題，但讀者容易忽略 `互動選項` 的存在（DB05 文案撰寫指南有提到）。建議在 DB05 段補一句說明。

（修訂：表單類型本身選項 ✅ 對齊。此條降為 🟡）

#### 3. **DB06 大分類欄位「明細類型」選項齊備但 schema 同時保留 `表單類型`/`登記類型` 等舊欄位**
指南：明細類型 7 選項（報名登記／聯繫互動／庫存紀錄／資料參考／圖文影音／細部流程／貨項進出）— **完全對齊** schema ✅
但 DB06 schema 仍有以下「孤兒」欄位干擾：
- `表單類型` select：`報名表單 / 出貨單`（疑似舊版未清理）
- `互動選項` select：`內部共識 / 聯繫互動 / 行政事務`
- `登記類型` select：`預約報名 / 意見回饋`
- `聯繫類型`、`紀錄類型`、`行政類型` 等多重小分類欄位
這不是指南錯，但指南只談「明細類型」會讓讀者誤以為 DB06 只有單一分類軸。建議補一句：DB06 schema 仍有歷史殘留分類欄位，AI 操作以 `明細類型` 為主。

#### 4. **DB02 大分類欄位「專案類型」實際 schema 中不存在**
指南第四節：
> DB02 專案績效管考｜專案類型｜地方文化事務 ／ 文化體驗服務

DB02 schema 中沒有「專案類型」select 欄位。最接近的是 `對應類型` select：`對應民間 / 對應公部門 / 對應自提`。
**這是致命錯誤**：指南把 DB02 大分類定義為「地方文化事務／文化體驗服務」（X/Y 軸別），但 schema 根本沒此欄位。要嘛新增此欄位，要嘛將指南改為對齊 `對應類型`（語意完全不同）。

#### 5. **DB05 名稱在第五節定義表寫成「登記表單明細」與其他段落「登記表單」不一致**

#### 8. 第三節日期屬性表「截止日期 / 實際執行日期」名稱與 schema 不符
schema 中對應欄位實際是 `截止時間` 與 `執行時間`（DB01~DB05 共用），且 DB03 還有 `預計交接日期`（rollup）。指南用「截止日期 / 實際執行日期」是抽象描述，建議補一句「對應 schema 欄位為 `截止時間` 與 `執行時間`」。

#### 9. 第三節「DB 06、07 不設截止日期或執行日期欄位」描述
DB06 schema 實際上有 `起算日期`（date）；DB07 沒有日期欄位 ✅。指南說 DB06「不設截止/執行日期」屬實，但讀者可能誤以為 DB06 完全沒有日期欄位。建議補註「DB06 仍有 `起算日期` 作為單時點欄位」。

#### 10. 第二節 DB05 收入欄位名「小計／總計」與 schema 不全對應
DB05 實際 schema 有 `收費小計`（formula）、`支出小計`（formula）、`預算總計`（formula）— 沒有單純的「小計」「總計」欄位。建議把指南調整為「`收費小計` / `預算總計`」。
DB06 也有 `小計`（formula）。

#### 11. 第六節三道欄位 — 「對應標籤」在 DB05 schema 中不存在
指南宣稱「對應標籤（relation → DB08）」是跨 DB 通用的第二道欄位。
- DB01: 有 `對應提案標籤`（→DB08）✅ 但名字不叫「對應標籤」
- DB02: 有 `對應標籤` ✅
- DB03: 有 `對應標籤` ✅
- DB04: schema 中**未直接看到** `對應標籤` 欄位，最接近的是 `ai_對應標籤`、`參考關聯`
- DB05: schema 中**沒有** `對應標籤` 欄位 — 但有 `對應對象`（→DB08）。CLAUDE.md 也指出「DB05 連結 DB08 用『對應對象』」
- DB06: 有 `對應標籤`、`對應內容標籤`、`對應標籤對象` ✅
- DB07: 有 `ai_對應標籤`（沒有非 ai 版本）
- DB08: 有 `ai_對應標籤`、自身的「自對標籤」（CLAUDE.md 描述但 schema 我沒明確看到 `自對標籤`，需另查）
- DB09: 有 `對應日期標籤`

**結論**：指南把「對應標籤」描述為跨 DB 通用欄位是過於簡化。實務上每個 DB 連 DB08 用的 relation 名字都不同。建議修正為「對應 DB08 的 relation（各 DB 名稱不同：DB05 為 `對應對象`，DB02/03/06 為 `對應標籤`，DB01 為 `對應提案標籤`）」。
