## 發現的問題清單

### 🔴 致命

1. **DB06 維護指南整份過時** (`3309ff25fdab8133b966c9c263cc7b3a`)
   - 欄位名稱錯誤：使用「異動類型」select，而現行 DB06 schema 是「明細類型」（select：報名登記/聯繫互動/庫存紀錄/資料參考/圖文影音/細部流程/貨項進出）。
   - 「異動類型」選項列出進貨/出貨/報名費/加購/盤點/退貨/調撥 — 這些其實是 DB05「庫存細項」+ DB06 業務情境的混合，與現行 schema 不符。
   - 「對應庫存」欄位名稱錯誤：DB07 的 relation 在 DB06 應透過「對應庫存→DB07」，但本指南把它當必填，且未提到「對應標籤對象」（DB06 連 DB08 用此欄位，不是「對應對象」）。
   - 庫存異動規則寫法不對：CLAUDE.md 與 DB05 維護指南都明確「DB05 庫存細項=進貨/出貨/盤點 → DB06 明細類型=庫存紀錄」，本指南未呈現。
   - 整份停留在 2026-03 版本，需重寫對齊現行 schema。

2. **DB07 維護指南大量 WP/WooCommerce 殘留** (`27e9ff25fdab802ab640d1e65f59ed1f`)
   - 第 2026-02-18 段、第 2026-03-23「WP 官網同步」段仍以 WooCommerce 為同步目標、提到 webhook 到 n8n 推 WC 產品。makesense.site WP 已退役，現行走 Supabase + Vercel。
   - FAQ Q6「WooCommerce 同步失敗怎麼辦？」整題過時。
   - 「DB04 控制欄位規範」段提到 `登記發佈/實際單價/數量上限` 三欄推 WooCommerce — 與現行 DB04 + Supabase events 流程脫鉤。
   - DB07 商品分類選項列「商品/耗材/設備/數位/其他」，但 CLAUDE.md/feedback 與子頁 `3429...9675` 列「選書/選物/數位/獎禮品/加購折價/票券/其他」是 `商品選項`（不是 `庫存類別`）— 兩個分類欄位混用，指南未說清楚。

3. **DB05 維護指南 collection ID 格式不統一** (`2799ff25fdab8094b7d1e3fffd620f91`)
   - 標示 collection ID 為 `e5f14f05-6c7c-4b8a-8043-04eab598fd4d`，但 CLAUDE.md 的 DB05 collection ID 是 `28a667a9-ede1-466a-9f18-419da33a8810`。前者其實是 page ID（`e5f14f056c7c4b8a804304eab598fd4d`），不是 collection ID。會誤導 AI/n8n 用錯 ID。
   - 「資料庫名稱」寫「登記表單明細」是舊名，現行為「登記表單」。
   - 「DB04 共識交接協作」是舊名，現為「協作交接」。
