# Supabase RLS 稽核 2026-05-05

project_id: `zgwdomvauuxaxtgqqvrn`

## 總覽
- public schema 共 **34 張表**
- **34/34 RLS=ON**（全部開）
- **23 張表有 policy**，**11 張表無 policy**（= 只有 service_role 進得去，最嚴格）

## 分類與風險評估

### A. 公開讀取表（OK，policy 正確）
anon SELECT all 是預期行為：
- `articles`, `events`, `products`, `topics`, `persons`, `partners`, `staff`
- `db08_places`（已用 anon_read 命名，乾淨）
- `social_metrics`, `translations`, `space_bookings`(讀已預訂時段), `reviews`

### B. 高風險：anon 可讀私人資料
**這幾張表的 anon SELECT all policy 應收緊**：

| 表 | 現況 | 風險 | 建議 |
|----|------|------|------|
| `members` | anon SELECT all | 全會員 PII 外洩（email/phone/LINE userId） | 移除 anon SELECT；前端改走 admin API（目前 dashboard 已是） |
| `orders` | anon SELECT all | 所有人訂單外洩 | 移除 anon SELECT；走 admin |
| `order_items` | anon SELECT all | 同上 | 移除 anon SELECT；走 admin |
| `registrations` | anon SELECT/INSERT all | 報名者 PII 外洩 | 保留 INSERT，移除 anon SELECT |
| `wishlist` | anon SELECT/INSERT/**DELETE** all | 任何人可刪光別人 wishlist | **緊急**：限制為 user_id 比對（或全走 admin） |
| `encore_requests` | anon SELECT/INSERT all | 申請者 email/動機外洩 | 保留 INSERT，移除 anon SELECT |
| `page_views` | anon SELECT/INSERT all | 行為足跡外洩 | 保留 INSERT，移除 anon SELECT |
| `search_logs` | anon SELECT/INSERT all | 同上 | 保留 INSERT，移除 anon SELECT |

### C. 已嚴格（RLS=ON, 無 anon policy）— 不需動
這些表只能用 service_role 存取，前端讀取必須走 admin API：
- `partner_applications`, `partner_performance`
- `point_ledger`, `staff_activities`, `staff_task_children_cache`, `staff_tasks_cache`, `staff_uploads`
- `uploads`, `vendor_preorder_items`, `vendor_preorders`, `vendor_photos`
- `workbench_notifications`, `market_applications`, `line_message_log`

## 已執行的低風險改動
**無**。所有 B 類收緊都會影響現有前端功能（任何用 anon key 讀這些表的地方會變空），需先確認前端 query 路徑後再執行。

## 待使用者確認的高風險改動

### 必須先驗證前端是否走 admin
請確認以下檔案中對 `members / orders / order_items / registrations / wishlist / encore_requests` 的 `from(...)` 呼叫，**全部都用 `supabaseAdmin`** 而非 `supabase`（anon client）：
- `/dashboard/*`（profile, orders 顯示）
- `/api/*`（所有 API Route）
- 票券/活動報名查詢
- LINE webhook handler

### 收緊 SQL（驗證後再跑）
```sql
-- members
DROP POLICY "Allow public select members" ON members;

-- orders
DROP POLICY "Allow public select orders" ON orders;
DROP POLICY "Allow public select order_items" ON order_items;

-- registrations（保留 INSERT）
DROP POLICY "Allow public select registrations" ON registrations;

-- wishlist（緊急）— 全 service_role 化或改為 auth.uid() 比對
DROP POLICY "Allow anon select wishlist" ON wishlist;
DROP POLICY "Allow anon insert wishlist" ON wishlist;
DROP POLICY "Allow anon delete wishlist" ON wishlist;
-- 之後前端必須走 admin API 才能讀寫 wishlist

-- encore_requests（保留 INSERT）
DROP POLICY "encore_public_read" ON encore_requests;

-- page_views（保留 INSERT）
DROP POLICY "Allow anonymous select page_views" ON page_views;

-- search_logs（保留 INSERT）
DROP POLICY "Allow anonymous select search_logs" ON search_logs;
```

### 風險最高 → 最低排序
1. **wishlist DELETE** — 任何人現在可以刪光所有人的 wishlist。最該先改。
2. **members SELECT** — PII 外洩。
3. **orders / order_items SELECT** — 訂單與金額外洩。
4. **registrations SELECT** — 報名 PII。
5. **encore_requests / page_views / search_logs SELECT** — 行為 / email。

## 安全鐵律遵守
- 未 disable 任何 RLS
- 未刪任何 policy
- 未 DROP/TRUNCATE/DELETE
- 所有改動都需使用者明示授權後再執行
