# 提 Notion 頁面一律寫完整名稱 + 超連結

## 規則
提到任何 Notion 頁面、指南、文件,**永遠**寫成「完整名稱 + 超連結」格式,不要只寫編號或簡稱。

## 正確 vs 錯誤

❌ 錯誤(只寫編號):
> 4-2-4-1 嗨嗨分析驗收完成
> 詳見 4-1-1
> 對齊 2-2-1 規則

❌ 錯誤(只寫名字):
> 詳見嗨嗨家族導航地圖

✅ 正確(完整名稱 + 超連結):
> [4-2-4-1 嗨嗨分析 原廠設定](https://www.notion.so/049/4-2-4-1-3639ff25fdab817c9ca5e4ec73dfbd2c) 驗收完成
> 詳見 [4-1-1 嗨嗨家族導航地圖](https://www.notion.so/049/...)
> 對齊 [2-2-1 欄位名稱指南](https://www.notion.so/049/32b9ff25fdab81caadd5e016cced1efe)

## 適用範圍
- 所有 4-X-X、3-X-X、2-X-X、1-X 編號的指南頁
- DB01~DB09 的 page 連結(雖然 DB 本身可以講 DB05 短名)
- 任何官網維護指南、嗨嗨家族文件

## 反面:該編號的地方不要漏編號
- 提「規則檔」「memory」要寫具體路徑(`.claude/MEMORY.md` 或 `.claude/memory/feedback_xxx.md`)
- 不要只說「memory」「記憶系統」

## 為什麼這條重要
四九已**多次**要求,會很煩。每次提到指南頁卻只給編號,他就要自己去找連結,等於浪費他時間。
