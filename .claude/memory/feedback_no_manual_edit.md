永遠不要叫用戶「搜尋這行換成那行」或「手動改一行」。

**Why:** 已經多次因為局部修改導致 heredoc 語法壞掉、CSS 截斷、函數消失、網站白屏。用戶不是工程師，手動改 PHP 非常容易出錯。

**How to apply:**
1. 修改 Code Snippets 時，永遠生成完整的 `.code-snippets.json` 匯入檔讓用戶直接匯入
2. 如果要改「額外的 CSS」，永遠給完整內容讓用戶清空貼上
3. 絕對不要說「搜尋 X 換成 Y」、「找到這行改掉」、「加在某行後面」
4. 程式碼太長無法在對話中完整顯示時，用 Write 工具輸出到檔案，再生成匯入用 JSON
