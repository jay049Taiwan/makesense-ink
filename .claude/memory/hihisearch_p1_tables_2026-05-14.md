## 表 4 — D 模式 registry pattern（DB07 16 商品類）

D 模式定義：對 1 個對象（商品／人物／關鍵字）為中心，廣撈 ≥10 筆不重複參考資料全存 DB06。

**registry 元素**：對象類型 → 搜尋語料模板 + 來源組合 + 至少撈幾筆 + DB06 寫入

### 選書類 4 種

**書籍刊物**
- 搜尋語料：`{書名}` + `{作者}` + ISBN + `{書名} 書評` + `{書名} 心得` + `{作者} 訪談` + `{作者} 其他作品`
- 來源：出版社官網、博客來/讀冊/誠品、Goodreads、Medium/部落格書評、媒體報導、圖書館目錄（國圖/大學圖書館）
- 至少：書本身 1、出版社 1、作者 1、書評 ≥3、相關作品 ≥2、訪談 ≥1、媒體 ≥1 = **10 筆**

**地圖海報**
- 搜尋語料：`{標題}` + 設計師 + 發行單位 + `{主題} 地圖` / `{主題} 海報`
- 來源：發行單位官網、設計師作品集（Behance/IG）、Pinterest 同類視覺、媒體報導、用戶評論
- 至少：商品 1、設計師 1、發行 1、同類視覺 ≥3、評論 ≥2、用法案例 ≥2 = **10 筆**

**筆記本 / 明信片**（流程相同）
- 搜尋語料：品牌 + 系列 + 材質 + 設計師
- 來源：品牌官網、設計師作品、用戶開箱、購買通路、同類商品
- 至少 10 筆

### 選物類 9 種

**生活用品 / 居家雜貨 / 文具玩具 / 隨身收納**（日常品共用模板）
- 搜尋語料：商品名 + 品牌 + 材質 + 用途 + 評價
- 來源：品牌官網、用戶評論（Mobile01/PTT/Dcard）、開箱文（YT/小紅書）、競品、購買通路（誠品/PChome/momo）、達人推薦
- 至少：商品 1、品牌 1、評論 ≥3、開箱 ≥2、競品 ≥2、推薦 ≥1 = **10 筆**

**身體用品**（特別注意成分認證）
- 搜尋語料：商品 + 品牌 + 成分 + 認證標章 + 過敏原 + 評論
- 來源：品牌官網、衛福部、用戶評論、開箱、競品、皮膚科部落格
- 必撈：成分表 1、認證 1（如有）、過敏警示 1（如有）

**植物**
- 搜尋語料：植物名 + 學名 + 養護 + 適應環境
- 來源：植物百科（維基/特生中心）、養護指南（園藝部落格）、品種介紹、相關植物、達人經驗
- 必撈：植物 1、學名/分類 1、養護指南 ≥2、相關品種 ≥2

**數位影音 / 藝術品**
- 搜尋語料：作品名 + 創作者 + 風格 + 評論 + 相關作品
- 來源：創作者官網/社群、平台（Bandcamp/Spotify/藝廊）、評論、其他作品、訪談、媒體報導
- 至少 10 筆（含相關作品 ≥3）

**服裝飾品**
- 搜尋語料：品牌 + 系列 + 材質 + 搭配 + 評論
- 來源：品牌官網、穿搭部落格、IG/小紅書、用戶評論、競品、達人推薦
- 至少 10 筆

### 數位類 3 種

**數位內容**（podcast/影集/電子書）
- 搜尋語料：內容名 + 創作者 + 主題 + 評論 + 同類
- 來源：內容平台、創作者背景、評論、相關內容、媒體報導
- 至少 10 筆

**數位貼圖**
- 搜尋語料：貼圖名 + 創作者 + LINE 商店 ID + 其他作品
- 來源：LINE 貼圖商店、創作者 IG/Twitter、UGC 截圖、相關作品
- 至少：貼圖 1、創作者 1、其他作品 ≥3、UGC ≥3、媒體 ≥1 = **10 筆**

**數位票券**
- **不介入**：票券是活動 sub-item，不獨立廣撈。對應活動才走 DB05 路線 C 活動推廣 採集。

## 卡住點（要請 四九 定調）

1. **書籍刊物的「圖書館目錄」要不要納**：國圖、各大學圖書館有書目資料，但對 makesense 業務（賣書/選書介紹）幫助有限。要嗎？
2. **「同類商品」如何定義**：D registry 寫「相關作品 ≥2」「同類視覺 ≥3」，相似度怎麼判？嗨嗨搜查不判斷，但「同類」這個篩選本身就是判斷。要不要由嗨嗨分析事後篩，搜查只負責量撈？
3. **C 模式社群數據的「對應內容」relation**：每天的數據要連到哪個 DB05？目前每平台應該有一個母 page（如「FB 粉專監測」），但這個母 page 還沒有實際的 Notion entity。要先建嗎？
