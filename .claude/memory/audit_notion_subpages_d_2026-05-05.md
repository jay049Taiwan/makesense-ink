## 六、DB07 庫存決策樹（3569ff25fdab812d8d71d113350c4e4c）

### 嚴重
17. **CLAUDE.md/MEMORY 明確規定：`庫存類型` = 商品 / 耗材 / 設備（與 `商品選項` 正交）**，但決策樹用 `庫存類型` 作第一層直接分支到 A 商品 / B 耗材 / C 設備，把「正交」誤寫成樹狀分流。實際應 `商品選項=選書/選物/數位` 在「商品」庫存類型下；耗材分流要看 `耗材選項`；但設計初衷是兩軸正交（一筆庫存可能同時 商品 + 自營，亦可能 耗材 + 進貨）——**樹形決策樹會誤導建檔**。
18. **「票券放在 數位 → 數位細項=票券」與 MEMORY 一致 ✅**；但 `數位細項` 此處列「內容 / 貼圖 / 票券」3 種，需驗 schema 是否還有其他選項。
19. **耗材選項「工作辦公 / 包裝零件 / 獎禮贈品」與 MEMORY 一致 ✅**

### 中度
20. C 設備「設備選項」列「辦公設備 / 活動設備 / 餐飲設備 / 陳列收藏」——但 MEMORY 寫「庫存類型 select：商品 / 耗材 / 設備」並提「辦公設備屬性 / 餐飲設備屬性」，未提「活動設備 / 陳列收藏」是否在 select 中。需驗 schema。
21. 多處 OCR / 簡繁誤字：「膆帶貼納」「掳刷抜擦」「達品」「卵脚」「渡道」（疑為早期翻譯錯字），影響 AI 判讀準確。
