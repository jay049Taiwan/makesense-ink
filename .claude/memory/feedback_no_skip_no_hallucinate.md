嗨嗨家族執行三鐵律（2026/05/14 訂定）。

## 鐵律 1：載 skill 後必須先 fetch 規範

載入任何嗨嗨 skill 後，動手前必須完成：
- fetch skill 內提到的所有 URL（工作導覽地圖、範本頁、指南頁）
- 列「動手前 checklist」回報給 四九：本項所有規範動作 → 四九 說「過」才能動

**Why**：我之前載 hihiadmin skill 後沒讀工作導覽地圖、沒 fetch 範本頁，憑印象做，導致 placeholder 沒套範本、沒建 view、沒檢查 對應執行日期 (DB09)。

**How to apply**：載 skill → 立即 fetch 該 skill 提到的所有外部 URL → 把每條規範動作列成 checklist → 等 四九 verify → 才動。

## 鐵律 2：遇到不知道的細節必須停下問

任何「具名人物／日期／數字／場景／引用文字」若無資料源 → 停下問 四九 → 不准腦補編造。

**Why**：我之前在 2024-01 月刊樣板裡編「籌備大半年」「2023 下半年籌備」「文學館主展廳改造」「開幕當天人比平常多」「特展是文學館存在理由的完整表述」「moku 開始準備春節限定」「年節文學主推」全是幻想。在 DB06 裡編「FB 貼文預估 ~25-30 篇」「文學館粉專 ~20-25 篇」也是憑空估的數字。

**How to apply**：寫任何具體細節前先問自己「這個資料源在哪？」找不到 → 停下問 四九，標「[待 四九 補]」是次選；最差也不能假裝知道。

## 鐵律 3：不准自決簡化／跳過規範

任何「跳過 / 簡化 / 不做 / 留待後續 / 採折衷方案」決策必須先問 四九。每篇做完列「規範對照表」（✅/❌）。

**Why**：我之前自決「不另建 DB06」「執行備註只標範圍指引不建素材」「跳過 4 個欄位（原文 URL／引用／對應連結／實際資料源）」這些都是自己壓縮路徑，被 四九 指正才補。

**How to apply**：
- 任何遇到「太貴／太久／規範模糊」想簡化 → 立即問 四九
- 每篇做完報「規範對照表」：條列規範第 1-N 條，逐條標 ✅/❌
- 失敗條目必須報原因（資料缺／時間／token 預算），等 四九 決定
