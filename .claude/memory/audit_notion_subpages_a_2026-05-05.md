## 問題清單

### 🔴 致命

**F1. 全部 4 份「成果報告/提案計畫 撰寫＋維護」指南仍寫 `文書類型`（已改名「文書備項」）**
- 主指南近期已將欄位由「文書類型」→「文書備項」+ 新增「新聞稿」。
- 但下列 4 份子頁的「識別條件」表格全寫 `文書類型 = 成果報告 / 提案申請`：
  - 成果報告撰寫指南（`3329ff25fdab819f8f78f59972d7c6ea`）
  - 提案計畫撰寫指南（`3329ff25fdab8125a41be0f443b4e9dc`）
  - 成果報告維護指南（`3329ff25fdab8141be84d5108a8583bd`，標題還寫「2026/04/22 更正」誤導）
  - 提案計畫維護指南（`3329ff25fdab81bdb0e1ffdc51ef68c3`，同上）
- 這 4 頁是 AI 觸發判讀核心，欄位名錯會導致整條 workflow 抓不到資料。

**F2. 定型通知撰寫指南：欄位 `聯繫細項=提醒通知` / `通知細項` schema 真實存在性未驗證**
- `3329ff25fdab81feba70e20922ee6386` 全文以「`互動選項=外部聯繫` + `聯繫細項=提醒通知` + `通知細項`（5 個 enum）」為入口判讀。
- DB05 schema（本地 MEMORY 與 CLAUDE.md）只列「互動選項=內部共識/外部聯繫」，未列出「聯繫細項」「通知細項」。如果 DB05 沒有這兩個欄位，整份指南無法落地（需與 DB schema 比對確認）。

**F3. 提案計畫撰寫指南引用「平安順興、宜日漫步」等品牌名做語氣校準**
- `3329ff25fdab8125a41be0f443b4e9dc` 第 4 步明列 `發佈單位 = 旅人書店、平安順興、宜日漫步`，並把「平安順興 / 宜日漫步」當作主動引用品牌之一。
- 與品牌敘事底稿（`3329ff25fdab812db998dac683d9d292`）列示的子品牌（旅人書店、平安順興、蘭博旅人、宜日漫步、森本屋）相對齊；但 MEMORY 中的核心品牌只列「旅人書店、moku」。需確認哪個是現行真相，否則 AI 生成的提案會張冠李戴。
