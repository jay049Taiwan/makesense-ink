跑某一工項（管考/搜查/聯想/分析/企劃/文案）時，如果意外發現屬於本職範圍的資料（例如搜查時發現一個 DB04 底本），**必須當場補進該頁的對應欄位**。不可以說「留給下一項處理」。

**Why**：下一項的職責是別的（搜查找底本、聯想找跨類引用、分析抽具名詞彙）。把本職資料推給下一項等於要求下一項重做你的工作，浪費 token 也違反 pipeline 設計。

**How to apply**：執行過程中發現的本職資料 → 立刻 update 該頁該欄位 → 之後才能設 ai_X=完成。如果是別項的職責資料，可以在自己的 執行備註 標註「⚠️ 建議下一項留意 X」做交接，但本項範圍內的不能跳過。

**對 四九 的影響**：不會出現「跑完搜查結果一半底本沒填，聯想進場時還要倒回去補」這種錯漏。
