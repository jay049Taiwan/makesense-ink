兩條規則分開記，不要混為一談：

1. **「對應標籤」(relation → DB08) — 家族都可寫**：任何嗨嗨家族成員跑任何 DB 時，若判斷對象/議題已有對應 DB08 page，直接加進「對應標籤」。這是連既有 page，不是建新 page。
2. **新建 DB08 page — 只有嗨嗨搜查、嗨嗨分析可以**：其他成員（聯想/企劃/文案/檢核/總管）一律不可新增 DB08 page。

**Why:** 「對應標籤誰看到誰補」是四九拍板的（鎖死太煩、會卡流程）。但 DB08 新建若也全開，會造成重複建檔與品質失控，所以 2026/05/16 四九明確收斂：DB08 新建權限只給搜查、分析兩項。

**How to apply:**
- 任何成員發現該掛的 DB08 標籤已存在 → 直接補進「對應標籤」。
- 發現對象/議題在 DB08 找不到（比對 名稱 + 同義備註 後）：
  - 若你是嗨嗨搜查或嗨嗨分析 → 可新建 DB08 page（先設 **經營類型=紀錄**；分析 pipeline 另由 DB06 觸發 page 啟動，不在此設退役欄位）。
  - 若你是其他工項 → **不要自己建**，在自己的 `執行備註` 標記「DB08 待新建：XXX」留給搜查/分析處理（`分析備註` 為分析專屬，其他棒不可寫）。
- 舊版指南若寫「對應標籤 = 嗨嗨聯想專屬」、或「DB08 新建 = 家族都可」一律過時。
