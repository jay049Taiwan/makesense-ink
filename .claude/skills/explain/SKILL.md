# /explain — 把話講清楚

被觸發後，把目標內容用「更好的語言」重新表達一次。目標：讓四九讀起來清楚、不困惑。

## 目標內容判定

1. 若 `/explain` 後面**有接內容** → 重講那段內容。
2. 若 `/explain` 後面**留空** → 重講 Claude 自己的**上一則回覆**。

## 重講規則（務必全部遵守）

1. **語言**：一律繁體中文，語法通順、句子完整。不要中英混雜到讓人讀不懂。
2. **英文標註**：凡是出現英文單字 / 專有名詞 / 技術術語，後面立刻用括號附上中文，例如：
   - `dog(狗)`
   - `cache(快取)`
   - `deploy(部署)`
   - `commit(提交)`
   - 已廣為人知、沒有合適中文、或本身就是專有代號的（如 `Notion`、`Supabase`、`GitHub`、檔名、變數名、指令）可以不附中文。
3. **解釋更清楚**：
   - 拆開冗長的句子，一句講一件事。
   - 把抽象的講法換成具體的（例如「優化效能」→「讓頁面開得更快」）。
   - 該舉例就舉例，該分點就分點。
   - 原文邏輯跳躍或語意含糊時，補上讓人看懂的脈絡。
4. **不要**：
   - 不要無意義的客套、奉承、贅字。
   - 不要簡體字。
   - 不要為了「更清楚」而加入原文沒有的事實或承諾。
   - 不要保留原文中文法不通、半英半中的句子。

直接輸出重講後的版本即可，不需要說明你做了什麼。
