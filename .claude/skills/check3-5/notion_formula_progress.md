## 資料檢核 formula 升級紀錄（2026-05-30）

**任務**：把逾期判斷整合進全 9 DB 的「資料檢核」formula。

**升級後公式邏輯**（DB01/09 done值="完成"；DB02-06 done值="已完成"；DB07/08 done值="已完成" OR "不處理"）：

```
if(empty(prop("TITLE")),
  [not-done check with overdue → "⚠️ 必填：TITLE [🔴 逾期 N 天] | routing:嗨嗨檢核"],
  [not-done check with overdue → "⚠️ [🔴 逾期 N 天] | routing:嗨嗨檢核" or ""]
)
```

逾期判斷規則：
1. 截止時間有值 → 看截止時間
2. 截止時間無值 + 執行時間有值 → 看執行時間
3. 兩者均無 → 不逾期

**技術筆記（重要）**：
- Notion Formula 2.0 的 `&&` 不 short-circuit — 日期欄位為 null 時兩側都會計算，造成 type error
- 解法：全用巢狀 `if()` 取代 `&&`，`empty()` guard 先判 null 再比較日期
- `let()` + 複雜日期計算組合也有相同問題（eager evaluation）
- 最終成功方案：純巢狀 `if()`，不用 `let()`，不用 `&&`/`||`
