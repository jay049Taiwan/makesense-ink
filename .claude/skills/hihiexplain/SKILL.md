# hihiexplain — 把上一段話翻得讓人聽得懂

四九 說「重講」「我聽不懂剛剛那段」「hihiexplain」「英文太多」時觸發，或四九直接貼上一段 AI 寫的文字。

四九 的常見情境：AI（多半就是你自己）剛剛那段解釋夾了一堆英文、用語繞、句子不通順，他不想再次自己腦補翻譯，要你重新講一次。

## 鐵律

1. **不新增資訊**：重寫只是換句話說，不要加入原文沒提到的承諾、行動、檔名、技術細節
2. **不刪資訊**：原文每一個重點都要保留，只是換成好懂的講法
3. **全程繁中**：除了專有名詞（如 `Next.js`、`Supabase`、`Cloudflare R2`、`Notion`、`Zeabur`、`Telegram`），其餘一律繁中
4. **英文補中文**：每個英文詞**第一次**出現時後面加 `（中文）`，第二次以後可不重複加
   - 範例：`bucket（儲存桶）`、`endpoint（連線端點）`、`webhook（網路鉤子）`
   - 已是台灣業界通用的英文不用翻：`API`、`URL`、`PDF`、`JSON`、`token`、`Git`、`commit`
5. **句子拆短**：原文一句話夾了三件事，就拆成三句
6. **動詞放前面，名詞別堆**：「為了讓 X 能夠 Y 的目的而進行 Z」改寫成「Z 之後 X 就會 Y」
7. **比喻換成日常生活**：例如「`volume（容量空間）` 是給 `container（容器）` 用的硬碟」可以直接說「就像幫程式準備一塊隨身硬碟，重開機後資料不會不見」
8. **保留結構**：原文有條列、表格，重寫版本也用條列、表格

## 流程

### 1. 確認對象

- 如果四九直接貼了一段文字 → 那段就是對象
- 如果四九只說「重講」「重新解釋」沒貼 → 對象是**你前一輪的回覆**
- 模糊時用一句話問：「是要我重講剛剛那段嗎？還是你想貼一段別人寫的？」

### 2. 解讀原意

- 先讀一遍抓主軸：這段在講「步驟」「決定」「狀態回報」還是「概念解釋」？
- 抓出原文每一個「事實點」（事實 = 不能改的具體資訊：檔名、數字、流程順序、依賴關係）
- 抓出原文用了哪些英文詞 → 列翻譯對照

### 3. 重寫

- 開頭直接講重點，不要鋪陳
- 中段依結構（條列 / 表格 / 段落）重寫
- 結尾告訴四九：「**接下來他要做什麼**」（如果原文有提到的話）

### 4. 對照檢查（內部，不輸出）

重寫完之後快速掃一次：
- ☐ 原文每個事實點都還在？
- ☐ 沒有自己加沒講過的承諾？
- ☐ 所有英文詞都補了中文（第一次出現）？
- ☐ 句子有沒有超過 25 字以上沒斷句的？

## 重寫的反面教材

| 別這樣 | 改這樣 |
|---|---|
| Deploy 完之後 service 會自動 restart | 部署（deploy）完之後，服務（service）會自己重啟（restart） |
| Webhook trigger 配合 schedule 跑 cron | 用 webhook（網路鉤子）觸發，搭配排程（schedule）跑定時任務（cron） |
| 把 endpoint 設成 S3-compatible 的 R2 endpoint | 把連線網址（endpoint）設成相容 S3 規格的 R2 連線網址 |
| 環境變數對齊 | 環境變數的名稱要對得上 |
| 上述需 root directory 設為 services/brand_monitor 否則會打到外層 Next.js 而失敗 | 注意要把根目錄（root directory）設成 `services/brand_monitor`。不然 Zeabur 會以為要部署外面那個 Next.js 官網，就會失敗 |
| D 段 / E 段 還在「無動靜」階段 | 每日簡報那封信裡的「D. 官網文章」「E. 官網活動」這兩節還顯示「昨日無動靜」 |

## 縮寫 / 編號 / 代號絕不可裸用

四九 看到 `A 段`、`DB05`、`MK0`、`partner_metrics_v`、`hihigm`、`L5` 這種詞，**如果這一輪對話沒有先講過它是什麼**，就一定要在重寫時補上脈絡。

範例：
- ❌ `A 段` → ✅ `每日簡報那封信裡的「A. 社群發文」那一節`
- ❌ `DB05 三層 relation` → ✅ `Notion DB05（登記內容）的三層關聯欄位（對應X / X引用 / X被引）`
- ❌ `partner_metrics_v` → ✅ `Supabase 裡那個合作夥伴績效統計的 VIEW（叫 partner_metrics_v）`
- ❌ `hihigm 巡查` → ✅ `嗨嗨總管（Telegram bot 名稱 hihigm）的例行巡查任務`

判斷方法：四九 把這段話複製到別的對話裡能不能看懂？看不懂就要補脈絡。

## 範圍邊界

- **不解答新問題** — 只是翻譯。四九問新東西時，請他另起一句正常問
- **不主動修錯** — 如果發現原文有事實錯誤，**先重寫成通順版**，最後加一行「⚠️ 順帶一提：原文這句『XXX』我重看覺得可能有錯，要不要我重新查？」
- **不下決定** — 原文留了選項給四九，重寫版也要留選項，不要替他選一個

## 互動風格

- 開頭一句話確認對象，避免重寫錯目標
- 結尾不加廢話結語（不要寫「希望這樣比較清楚！」「有任何問題歡迎再問」）
- 全程符合 noah-communication-style 的風格規範
